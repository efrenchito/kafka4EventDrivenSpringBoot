🚀 Producing Kafka Message without a Key
>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
Before trying to create a Kafka Topic ensure Kafka Server is up and running
[terminal]
$ ./bin/kafka-server-start.sh config/kraft/server-1.properties
$ ./bin/kafka-server-start.sh config/kraft/server-2.properties
$ ./bin/kafka-server-start.sh config/kraft/server-3.properties
...
[terminal]
$ ./bin/kafka-console-producer.sh --bootstrap-server localhost:9092,localhost:9094 --topic 💥my-topic

🧨⚠️🤯 If you send a message to a Topic that doesn't exist
 -> Kafka Producer will respond with error
 -> However if property 'auto.create.topics.enable' is set to true, it will also automatically create the topic
=======================================================================================================================================

[terminal]
$ cd {WORKSPACE}
$ cd kafka/bin
$ ls
... kafka-topics.sh  💥kafka-console-producer.sh

$ ./bin/kafka-console-producer.sh --bootstrap-server localhost:9092,localhost:9094 --topic 💥my-topic

> Hello World 2

🧐🕵️🔎 Even though I got that error message, I could still continue sending messages
 -> By default Kafka is configured to automatically create a new topic if that topic does NOT exist
 => This is defined at the property  

🧨⚠️🤯 If you send a message to a Topic that doesn't exist
 -> Kafka Producer will respond with error, but it will also automatically create the topic


🧐🕵️🔎 Even though the topic can be created automatically in case it doesn't exist a best practice is 
 -> To create the Topics before sending messages to it 
 -> Otherwise the Topic will be created with the default Configuration and that could not match with what you need

🚀 Producing Kafka Message with a Key
>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
🧠 --property "parse.key=true" --property "key.separator=:"
~
🧐🕵️🔎 Kafka messages sent with a Key, will be stored in the same partition
 -> Ensuring they are stored in order.
..
[terminal]
$ ./bin/kafka-console-producer.sh --bootstrap-server localhost:9092,localhost:9094 --topic 💥my-topic
--property "parse.key=true" --property "key.separator=:"

🧐🕵️🔎 When using --property "parse.key=true" Producer expects messages to be in the format of key-value pair
Then we must specify the separator character via --property "key.separator=:"
=======================================================================================================================================
In this lesson, we'll learn how to send a message with a key to Kafka Topic
Messages and Kafka Topics are stored as Key value pairs.
When sending messages without a Key, then key will be null.

Kafka messages sent with a Key, will be stored in the same partition
 -> Ensuring the are stored in order.

When you need to store and read messages in order, you need to use the same key for those messages


When using --property "parse.key=true" Producer expects messages to be in the format of key-value pair
Then we must specify the separator character via --property "key.separator=:"


$ ./bin/kafka-console-producer.sh --bootstrap-server localhost:9092,localhost:9094 --topic 💥my-topic
--property "parse.key=true" --property "key.separator=:"
> firstName:Sergey 

If we try to send a message without the separator we'll get an Error Message
> test message  => No key separator found
